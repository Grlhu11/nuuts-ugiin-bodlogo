#  Өгөгдлийн Сангийн Нууц Үгийн Бодлого (Database Password Policy)

Энэхүү баримт бичиг нь байгууллагын өгөгдлийн сангийн аюулгүй байдлыг хангах, зөвшөөрөлгүй хандалтаас сэргийлэх зорилгоор боловсруулагдсан нууц үгийн стандартыг тодорхойлно.

## 1. Нууц үгийн нарийн төвөгтэй байдал (Complexity Requirements)
Өгөгдлийн санд бүртгэлтэй бүх хэрэглэгчид дараах шаардлагыг хангасан нууц үг ашиглана:
* **Урт:** Хамгийн багадаа **12 тэмдэгт**.
* **Бүтэц:** Том үсэг (A-Z), жижиг үсэг (a-z), тоо (0-9) болон тусгай тэмдэгтүүд (!@#$%^&*) заавал орсон байх.
* **Хориглох зүйл:** Хэрэглэгчийн нэр (username), компанийн нэр, эсвэл "Password123" гэх мэт нийтлэг үгсийг ашиглахгүй байх.

## 2. Нууц үгийн мөчлөгийн удирдлага (Password Lifecycle)
* **Хүчинтэй хугацаа:** Нууц үгийг **90 хоног** тутамд заавал шинэчилнэ.
* **Дахин ашиглалт:** Сүүлийн **10 удаа** ашигласан нууц үгийг дахин ашиглахыг хязгаарлана.
* **Шинэчлэлт:** Нууц үгийг сольсноос хойш 24 цагийн дотор дахин солихыг хориглоно (хуучин нууц үг рүүгээ хурдан буцахаас сэргийлэх).

## 3. Хандалтын хязгаарлалт (Account Lockout Policy)
* **Буруу оролдого:** Дараалсан **5 удаа** буруу нууц үг оруулбал тухайн дансыг **30 минутын** хугацаатай автоматаар түгжинэ.
* **Администраторын эрх:** Системийн администраторууд (DBA) болон өндөр эрхтэй хэрэглэгчид заавал **Хоёр шатлалт баталгаажуулалт (2FA/MFA)** ашиглана.

## 4. Хадгалалт ба Нууцлал (Technical Implementation)
* **Hashing:** Нууц үгийг өгөгдлийн санд хэзээ ч ил текстээр хадгалахгүй. Заавал **Argon2** эсвэл **bcrypt** алгоритм ашиглаж hash-лах ёстой.
* **Salting:** Солонгосон халдлагаас (Rainbow table attacks) сэргийлэхийн тулд нууц үг бүрт өвөрмөц **Salt** утга нэмж хадгална.
